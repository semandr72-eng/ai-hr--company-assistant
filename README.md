# 🤖 AI HR Ассистент

Готовый AI-ассистент для сотрудников и специалистов отдела по работе с персоналом. Построен на современной **RAG-архитектуре** (Retrieval-Augmented Generation) с кэшированием ответов.

## ✨ Функционал

- **RAG-пайплайн** — поиск по корпоративной базе знаний и генерация ответов на основе найденной информации
- **Кэширование ответов** — уменьшение нагрузки на LLM и ускорение повторных запросов
- **Векторное хранилище** — ChromaDB для эффективного поиска по семантическим Embedding
- **Поддержка LLM** — OpenAI (GPT) и GigaChat (российская модель)
- **Многофайловая поддержка** — обработка различных типов документов
- **Оценка качества** — интеграция с RAGAS для метрик Faithfulness и Answer Relevance

## 🚀 Быстрый старт

### Требования

- Python 3.11+
- API ключ OpenAI или GigaChat
- Git

### Установка

```bash
# 1. Клонируйте репозиторий
git clone <your-repo-url>
cd assistant_api

# 2. Создайте виртуальное окружение
python -m venv venv_py311

# 3. Активируйте виртуальное окружение
# Windows:
venv_py311\Scripts\activate
# Linux/Mac:
source venv_py311/bin/activate

# 4. Установите зависимости
pip install -r requirements.txt

# 5. Настройте переменные окружения
copy .env.example .env    # Windows
# или
cp .env.example .env      # Linux/Mac

# 6. Заполните .env своими ключами
```

## ⚙️ Настройка окружения

Скопируйте `.env.example` в `.env` и заполните значения:

```env
# GigaChat API
GIGACHAT_API_KEY=your_gigachat_api_key_here

# OpenAI
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-3.5-turbo

# ChromaDB
CHROMA_PERSIST_DIR=./chroma_db

# RAG настройки
CHUNK_SIZE=500
CHUNK_OVERLAP=50
EMBEDDING_MODEL=sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
TOP_K=5
```

## ▶️ Запуск

### Вариант 1: Assistant API (OpenAI)

```bash
# Перейдите в директорию
cd assistant_api

# Запустите сервер
python app.py
```

Сервер доступен по адресу: `http://localhost:8000`

### Вариант 2: Assistant Giga (GigaChat)

```bash
# Перейдите в директорию
cd assistant_giga

# Запустите сервер
python app.py
```

### Инициализация базы знаний

При первом запуске документы автоматически загрузятся в ChromaDB. Для переиндексации:

```bash
# Удалите старую базу
rm -rf chroma_db

# Запустите сервер заново
python app.py
```

## 📊 Оценка качества

Проект включает скрипт для оценки качества RAG-пайплайна с помощью RAGAS:

```bash
python evaluate_ragas.py
```

Метрики:
- **Faithfulness** — насколько ответ основан на найденных документах
- **Answer Relevance** — насколько ответ релевантен запросу

## 🔧 Разработка

### Добавление новых документов

Поместите документы в папку `assistant_api/data/` и перезапустите сервер.

### Изменение параметров RAG

Отредактируйте переменные в `.env`:
- `CHUNK_SIZE` — размер чанка при разбиении документов
- `CHUNK_OVERLAP` — перекрытие между чанками
- `TOP_K` — количество найденных похожих документов
- `EMBEDDING_MODEL` — модель для создания embedding
